# ServEase Home Service Platform - Project Report

## 1. Project Overview
ServEase is a comprehensive, full-stack home service booking platform that connects customers with verified service professionals. The platform allows users to browse services, add specific subservices to a cart, book appointments securely, and review their service experiences. Professionals have their own dedicated portal to manage bookings, track their schedule, and handle service completion securely using a unique PIN verification system.

## 2. Technology Stack
- **Frontend**: Next.js (App Router), React, Tailwind CSS
- **Backend**: Node.js, Express.js
- **Database**: PostgreSQL
- **Authentication**: JWT (JSON Web Tokens), bcryptjs for password hashing

## 3. Core Features & Capabilities

### Customer Experience
- **Dynamic Catalog**: Browse services by categories with a rich, animated UI.
- **Provider Selection**: View detailed provider profiles, including past customer reviews, experience, and pricing.
- **Cart & Checkout**: Add specific subservices to a cart, select dates and times, and complete a secure booking.
- **Customer Dashboard**: Track ongoing and past bookings.
- **Service Verification**: Receive a secure 6-digit PIN to share with the provider upon satisfactory service completion.
- **Rating & Reviews**: Rate providers out of 5 stars and leave written feedback to help the community.

### Provider Portal
- **Dedicated Authentication**: Separate secure login and registration flow for service professionals.
- **Provider Dashboard**: Manage incoming bookings, view customer details (location, phone), and update job statuses.
- **Secure Job Completion**: Providers must input the 6-digit PIN given by the customer to mark a job as "Completed", ensuring accountability and preventing fraudulent completions.
- **Review Tracking**: Providers can see feedback left by customers on their profile to track their performance.

## 4. UI/UX Design (Aesthetics)
The application is built with a highly premium, modern design language:
- **Glassmorphism**: Extensive use of blurred, translucent backgrounds, and frosted glass overlays.
- **Micro-Animations**: Smooth hover states, sleek transitions, animated pill navigations, and fade-ins.
- **Clean Typography**: High contrast, easily readable fonts with proper hierarchical structure.
- **Responsive Layouts**: Fully mobile-optimized layouts tailored for seamless use on all screen sizes.

## 5. Architecture & Database Schema
The system runs on a normalized PostgreSQL database to ensure data integrity and seamless relational mapping:
- **users**: Stores customer accounts and authentication data.
- **service_providers**: Stores professional accounts and profiles.
- **categories & services**: Hierarchical mapping of available services.
- **subservices**: Specific granular tasks (e.g., "AC Gas Refill" under "AC Repair").
- **provider_services**: Junction table mapping which provider offers which service, their specific base price, and years of experience.
- **provider_subservices**: Granular pricing by specific providers for specific tasks.
- **bookings**: Core transaction table tracking the customer, provider, total price, scheduled date, status (Pending, Accepted, Completed), and the highly secure 6-digit `completion_pin`.
- **reviews**: Links a customer, provider, and booking to a rating and text comment.
- **cart**: Persistent temporary storage for checkout flows across sessions.

## 6. Security & State Management
- **Token-Based Authentication**: Secure stateless JWTs are used for authentication and authorization.
- **Password Protection**: Passwords are securely hashed using bcryptjs.
- **Dynamic Routing**: Built-in logic routes authenticated users away from login screens and protects dashboard routes from unauthenticated access.
- **Transactional Integrity**: The backend utilizes careful relational mapping to ensure bookings cannot be made for invalid providers or subservices.

## 7. Setup & Run Instructions
1. Ensure PostgreSQL is running and the database credentials are configured in `backend/.env`.
2. Open a terminal in the `backend/` directory, run `npm install`, and then run `node index.js` to start the API server on `http://localhost:5000`.
3. Open a second terminal in the `frontend/` directory, run `npm install`, and then run `npm run dev` to start the Next.js development server on `http://localhost:3000`.
4. Open the application in your browser and explore!
